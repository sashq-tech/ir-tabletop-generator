# Content Expansion Backlog

The prototype should deepen the current five incident categories before adding new categories.

## Current Categories

- Ransomware disruption
- Credential phishing
- Insider data exposure
- Public service outage / DDoS
- Vendor compromise

## Per-Category Expansion Target

Each category should eventually have:

- 6-10 openings
- 8-12 initial facts and unknowns
- 5+ injects per phase
- 8-12 decision points
- 6-10 stakeholder messages
- 8-12 evidence needs
- 6-10 assumptions to validate
- Audience-specific facilitator notes

## Phase Inject Target

Current phases:

- Detect
- Triage
- Contain
- Communicate
- Recover

Each phase should include injects that force a different kind of discussion:

- Detect: first report, alert, ticket, vendor notice, user complaint
- Triage: conflicting facts, uncertain scope, missing logs, ownership confusion
- Contain: authority, business impact, evidence preservation, access changes
- Communicate: staff guidance, leadership brief, public status, partner/vendor notice
- Recover: restoration criteria, monitoring, follow-up ownership, residual risk
