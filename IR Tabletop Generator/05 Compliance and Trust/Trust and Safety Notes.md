# Trust and Safety Notes

## Core Disclaimer Direction

The tool should clearly state that it creates educational tabletop exercise materials and does not replace:

- An organization-specific incident response plan
- Legal advice
- Compliance review
- Cybersecurity consulting
- Internal policy
- Regulator or contractual requirements

## Data Handling Promise

The product should emphasize:

- No login
- No uploaded files
- No organizational secrets required
- No backend processing
- Generation happens in the browser

## Compliance Language Guardrails

Use careful language:

- "Supports tabletop preparation"
- "Helps organize exercise evidence"
- "Can help teams discuss common incident response decisions"
- "May be useful for internal readiness documentation"

Avoid overclaims:

- "Compliant with"
- "Audit guaranteed"
- "Meets all requirements"
- "Legally sufficient"
- "Official"

## Credibility Needs

- Explain what a tabletop exercise is.
- Explain how to use generated materials safely.
- Encourage customization to local policies and systems.
- Provide a source/reference page later.
- Make the print packet look serious and restrained.
